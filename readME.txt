change：

product   model添加两列属性，一个为publish，记录是哪一个卖家发布的，一个为repertory，记录该product的库存。

user   model添加了一列属性，为identity，记录该用户是卖家还是买家。

用户分为三类：
1.管理员
2.顾客：
	顾客登陆后，可以看到整个书店当前有多少书目提供购买。
	顾客点击Home页面，可以链接到书店的主页。

3.卖家：
	卖家登陆后，可以看到自己发布的产品的书目，以及接到订单的数目
	点击Home，可以看到书店的主页，但是每一本书下面没有了Add to cart，而是变成了如果是自己发布的书，就会有Edit和Delete的按钮。
	点击Orders，卖家就可以看到有自己所有接到的订单，并且可以对订单做相应的发货或者取消的操作。并且发货或取消后，会有相应的邮件信息发送给买家。
	点击Products，可以看到所有该卖家发布的产品，并且可以对产品进行Edit或者是delete的操作。
	卖家查看订单时，有send链接，可以进行发货处理，如果货物足够，则会有Send按钮，否则显示货物不足的信息，并且出现让卖家修改库存的按钮。
	卖家点击发货后，会向买家发送发货邮件，而如果卖家点击缺货按钮，则会发送缺货邮件给卖家，并让买家耐心等待。
