一、用户分类：

	1.买家：买家在注册时需要选择身份为买家，该属性记录在identity中，买家登陆后跳转到admin页面中，可以看到当前商店中书籍的数目。回到主页可以添加书籍进入购物车，确认购物车后，并填写相关订单信息，会收到订单确认邮件。买家点击orders还能查看自己的所有订单信息，并可以对未发货的订单做删除或修改的处理。

	2.卖家：卖家登录后，跳转到admin页面可以看到自己发布了的产品的数目以及自己还有多少个订单条目未处理。卖家点击productss可以看到自己发布的所有产品，并且可以添加、修改或删减自己发布的产品，点击orders可以看到所有有关于自己产品的订单条目信息。

	3.管理员：管理员登录后跳转到admin页面，可以看到商店产品数目，以及订单数目，用户数目以及订单条目总数目。回到主页，可以添加产品或对所有产品进行修改和删除。点击Orders，可以看到所有的订单，并可以对订单做增删改处理。点击Products，可以看到所有的产品，并可以对所有产品做增删改处理。点击Users，可以看到所有的用户，并可以对所有的用户进行增删改处理。点击Vision，可以看到所有未被处理的愿景，并可以对这些愿景做增删改处理。


二、库存与发货确认：
	1.买家：在其订单被发货后会收到相应的发货确认邮件。
	
	2.卖家：点击Orders后，可以看到所有订单条目，并且每一个条目后方都有一个发货按钮，点击发货按钮，进入发货页面，如果库存足够，卖家可以点击发货的按钮，点击了发货按钮，相应的条目被删除，并且发送发货确认邮件给买家。卖家也可以店家stockout缺货按钮，不会删除相应的条目，但会给买家发送缺货邮件。
	
	3.管理员：管理员可以对订单做删除的处理。

三、愿景Vision
	1.买家：如果有想要买而书城却没有的书籍，买家可以点击Vision，并且发布自己想要购买的书籍的信息。买家可以看到自己所有发布了的而未被处理的vision的信息，并可以对这些vision做删改的处理。

	2.卖家：卖家点击Vision后，可以看到所有买家发布的vision，并且可以选择自己心仪的产品进行产品发布。即点击vision后方的处理按钮。点击处理按钮后，系统会自动创建与该vision条目title一致的书籍，并跳转到该书籍的修改页面，以及发送愿景接受邮件给买家。

	3.管理员：管理员可以对vision做删除的处理。

	一.基于用户的购物车：
	
		1.购物车添加属性：user_id和user_name，
			身份是customer的用户才可以对自己的购物车，进行创建，编辑，删除，清空，下单的操作；
			身份是seller的用户只可以看到跟自己商品有关的购物车，不能修改和创建；
			身份是administor的用户可以查看所有的购物车，并只能删除，不能修改和创建。
			
		2.customer登录，访问主页，若购物车为空，自动创建一个新的空的购物车
			用户的当前购物车为最近创建一个购物车；删除当前购物车，自动找回上一个最新创建的购物车；创建新的购物车被当成当前购物车
			点击mycart页面，显示自己所有的购物车，可对每个购物车查看，编辑，删除，合并到当前购物车
			编辑页面，可移除某个商品，修改某个商品的数量，清空购物车，对任一购物车进行下单
			
		3.administor登录，点击mycart，查看所有customer的所有carts，操作权限只有删除。
		
		4.seller登录，点击mycart，看看所有与自己商品有关的购物车，不能修改，创建和删除。
